# User Flow Test Skill

## 목적
핵심 사용자 시나리오를 실제 브라우저에서 재현하고 녹화한다.

## 실행 절차

### Step 1: Recording 시작

### Step 2: 시나리오 실행
매 단계 완료 시 자동 스크린샷 캡처.

### Step 3: 검증 포인트
- 페이지 전환 정상 여부
- 로딩 상태 표시 (빈 화면 아닌지)
- 클릭 가능 요소 반응 여부
- 뒤로가기 시 상태 유지 (스크롤, 필터)
- 모바일 터치 크기 44px 이상

### Step 4: 보고
Recording → Artifact. 실패 단계는 스크린샷 + 예상 vs 실제.

## 기본 시나리오

### A. 독자 여정 (Public)
1. www.mhj.nz 홈 접속
2. 히어로 캐러셀 좌우 스크롤
3. Latest Posts에서 글 클릭
4. 블로그 상세 하단까지 스크롤
5. AI Insight 영역 확인
6. 뒤로가기 → 블로그 목록
7. 카테고리 필터 (Home Learning) 클릭
8. 필터 결과 확인
9. About 페이지 이동
10. 매거진 서가 → 이슈 → 기사 열기

### B. 뉴스레터 구독 (Public)
1. 홈에서 뉴스레터 CTA 찾기
2. 이메일 입력 (test@example.com)
3. 제출 클릭
4. 성공/에러 메시지 확인
5. 같은 이메일 재시도 → 중복 처리 확인

### C. Admin 기본 (MHJ Desk)
1. /mhj-desk 접속 → Google OAuth 로그인
2. 블로그 목록 확인
3. 새 블로그 작성 → TipTap 입력
4. 이미지 업로드
5. 미리보기 → 저장 (draft)
6. 뉴스레터 섹션 이동

### D. 매거진 뷰어 (Public)
1. /magazine 접속
2. 서가 좌우 스크롤
3. 이슈 클릭 → 기사 목록
4. 기사 클릭 → A4 모달
5. 이미지 라이트박스
6. 리액션 버튼 클릭
7. 모달 닫기

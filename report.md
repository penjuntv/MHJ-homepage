# MHJ Homepage 전체 디자인 검수 + 사이드 배너 조사 보고서

## Part A: 전체 페이지 디자인 검수 (Playwright 스크린샷 캡처 완료)

현재 웹사이트의 주요 접근 가능 페이지들에 대한 캡처를 완료했습니다. 데스크톱(1440px) 및 모바일(390px, iPhone 14 Pro 기준) 뷰포트에서 총 10장 이상의 스크린샷이 `screenshots/` 폴더 내에 저장되었습니다.

### 캡처 요약 및 이슈 검토 결과:
- **전체 공통**: Navigation 여백은 일관성을 보이고, 푸터도 정상적으로 렌더링되고 있습니다. 폰트(Playfair Display + Noto Sans KR) 또한 Playwright 렌더링을 기다려 확인한 결과 정상 로드되었고 가로 오버플로우는 발견되지 않았습니다.
- **랜딩 (`/`)**: 히어로 이미지 및 Intro 섹션의 데스크톱 2컬럼 / 모바일 1컬럼 반응형 전환이 안정적입니다.
- **About (`/about`)**: 가족 프로필 카드 반응형(모바일 1컬럼 그리드) 레이아웃이 요구사항에 맞게 잘 동작합니다.
- **매거진 (`/magazine`)**: 가로로 스크롤 가능한 서가 형태가 문제없이 표시됩니다.
- **블로그 라이브러리 (`/blog`)**: 필터 및 썸네일 카드 영역이 제 비율을 유지하며 표시됩니다. 
- **블로그 상세 (`/blog/[slug]`)**: 드롭캡과 인라인 이미지가 정상적으로 출력되며 좌우 여백의 일관성이 잘 지켜지고 있습니다.

## Part B: 사이드 배너 (Sticky Sidebar) 조사

### [공간 분석]
- **데스크톱 1440px 기준 콘텐츠 영역 너비**: `720px` (가운데 정렬)
- **데스크톱 1440px 기준 좌우 여백 공간**: 각 **`360px`** 
- **사이드바 배치 가능 여부**: **가능**
- **최소 뷰포트 기준**: **`1024px` 이상에서 가능**. 콘텐츠 720px + 적절한 사이드바 너비(약 200px) + 좌우 간격(약 40px)을 감안하면 최소 960px~1024px의 해상도 환경이 사이드바 유지에 적합합니다. 1024px 이하에서는 CSS 미디어 쿼리(`@media (max-width: 1024px)`)를 사용해 사이드바를 보이지 않게(`display: none`) 처리하는 것이 좋습니다.

### [현재 HTML 구조]
- **사이드바 추가 시 변경 필요 여부: 변경 필요**. 
현재 블로그 상세 페이지의 콘텐츠는 `maxWidth: 720px`, `margin: 0 auto`를 지닌 단일 `<div>`로 가운데 정렬되어 있습니다. 사이드바가 추가되려면 기존 `<div>`를 래핑하는 새로운 부모 요소(예: `display: flex; max-width: 1200px; margin: 0 auto; gap: 32px;`)를 추가하여, 왼쪽(혹은 오른쪽)에 `<aside>` 태그로 Sticky Sidebar를 넣고, 우측에 기존 720px 콘텐츠 컴포넌트가 위치하는 2-컬럼 구조로 DOM 트리를 수정해야 합니다.

### [사이드바 활용 아이디어]
1. **목차 (Table of Contents)**: 긴 블로그 에세이를 읽을 때 현재 어느 소제목 파트를 지나고 있는지 하이라이팅 처리 (`Intersection Observer` 활용).
2. **뉴스레터 & 즐겨찾기 (Newsletter & Social)**: 스크롤해도 항시 노출되는 뉴스레터 이메일 구독 폼과 SNS 공유 버튼 단축 모듈을 배치하여 전환율 상승 유도.
3. **최근 인기 글 (Featured/Popular Posts)**: 본문을 읽고 난 사용자가 바로 떠나지 않도록 사이드바에 다른 주요 블로그 글 3개 정도의 썸네일 링크를 스티키(Sticky)로 제공.

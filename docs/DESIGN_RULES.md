## 1. MHJ의 정체성

MHJ는 Kinfolk도 Cereal도 아니다.
- Kinfolk/Cereal은 **프로 사진가 + 프로 에디터**가 만드는 상업 매거진
- MHJ는 **가족이 직접 찍고 직접 쓰는 라이프 아카이브**

따라서:
- 너무 차갑게 절제하면 → 가족의 온기가 사라진다
- 너무 화려하게 장식하면 → 아마추어 블로그처럼 보인다
- **"따뜻하되 세련된, 감성적이되 지적인"** — 이것이 MHJ의 톤

MHJ가 추구하는 감각:
- 한국 감성의 따뜻함 + 에디토리얼 매거진의 구조적 절제
- 가족 사진의 친밀함 + 활자의 격조
- 콘텐츠의 밀도 + 여백의 여유

## 2. 레이아웃 컨테이너

### 2.1 이중 컨테이너 시스템

MHJ는 **풀-와이드(full-width)**와 **콘텐츠 영역(contained)**을 섹션 성격에 따라 병용한다.

| 용도 | max-width | 적용 대상 |
|------|-----------|-----------|
| **몰입형 섹션** | 100vw (풀-와이드) | 히어로 캐러셀, 매거진 서가, 대형 배경 이미지 |
| **콘텐츠 섹션** | 1320px | 블로그 그리드, About, 아티클 목록, 인트로 |
| **본문 읽기 영역** | 720px | 블로그 상세 본문, 매거진 아티클 본문 |

### 2.2 왜 1320px인가

- 1200px: 텍스트 중심 사이트에 적합 (Monocle, Apartamento)
- 1440px: 이미지 포트폴리오 사이트에 적합
- 1320px: MHJ는 텍스트와 이미지가 공존하고, 4열 카드 그리드에서 카드당 ~300px 확보에 최적
- 계산: 4열 × 300px + gap 24px × 3 = 1272px + padding 48px = 1320px

### 2.3 컨테이너 CSS

```css
.container-content {
  max-width: 1320px;
  margin: 0 auto;
  padding: 0 clamp(20px, 4vw, 48px);
}

.container-reading {
  max-width: 720px;
  margin: 0 auto;
  padding: 0 clamp(20px, 4vw, 32px);
}
```

### 2.4 울트라와이드 (2560px+) 대응

- 콘텐츠 영역: 1320px 유지, 중앙 정렬
- 풀-와이드 섹션: 이미지가 늘어남 — object-fit: cover로 처리
- 양쪽 남는 공간: 페이지 배경색(white/dark)

## 3. 그리드 시스템

### 3.1 반응형 열 변화

- < 640px (sm): 1열 — 모바일
- 640~1023px (md): 2열 — 태블릿
- 1024~1319px (lg): 3열 — 소형 데스크탑
- 1320px+ (xl): 4열 — 대형 데스크탑

### 3.2 gap 규칙

- 카드 간 gap: 24px (3 × 8px 그리드)
- 섹션 간 여백: 96px (12 × 8px 그리드)
- 카드 내부 텍스트 padding: 16px (2 × 8px 그리드)

### 3.3 Tailwind 적용 예시

```html
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 xl:grid-cols-4 gap-6">
```

## 4. 이미지 비율

### 4.1 기본 비율 체계

| 용도 | 비율 | 이유 |
|------|------|------|
| **블로그 카드** | 3:4 (세로) | 에디토리얼 표준, 사진 디테일 유지 |
| **히어로/배너** | 16:9 또는 21:9 | 몰입감, 풍경 활용 |
| **매거진 아티클 카드** | 3:4 (세로) | 매거진 페이지 느낌 |
| **About 프로필** | 4:5 (세로) | 인물 사진 최적화 |
| **갤러리/인스타** | 1:1 (정사각) | 썸네일 통일성 |
| **블로그 상세 본문 이미지** | 원본 비율 유지 | 콘텐츠 존중 |

### 4.2 가로/세로 혼합 금지

하나의 그리드 안에서 가로와 세로 비율을 섞지 않는다.
같은 섹션의 카드는 반드시 동일한 비율을 사용한다.

### 4.3 이미지 CSS

```css
.card-image {
  aspect-ratio: 3 / 4;
  object-fit: cover;
  width: 100%;
}
```

## 5. 타이포그래피

### 5.1 폰트 패밀리 (변경 없음)

- Display/Headings: Playfair Display (serif)
- Body/Label: Noto Sans KR (sans-serif)

### 5.2 크기 체계

| 용도 | 크기 | 비고 |
|------|------|------|
| **히어로 제목** | clamp(32px, 6vw, 90px) | 일반 슬라이드 / StoryPress: clamp(36px, 8vw, 120px) |
| **페이지 타이틀 (type-display)** | clamp(40px, 5vw, 56px) | --fs-display CSS 변수 |
| **섹션 제목 (type-h1)** | clamp(32px, 4vw, 48px) | --fs-h1 CSS 변수 |
| **카드 제목** | 18px (고정) | 크기 변동 없이 일정 |
| **본문** | 16px, line-height 1.7 | 한국어는 1.7이 최적 |
| **라벨/캡션** | 11px, letter-spacing 2px | uppercase, 회색 |
| **메타 정보** | 13px | 날짜, 저자 등 |

### 5.3 제목 크기 상한 원칙

- 일반 콘텐츠 제목: **56px을 초과하지 않는다** (type-display 기준)
- 히어로 캐러셀: 일반 슬라이드 **90px**, StoryPress **120px** 상한
- clamp() max값이 이 기준을 초과하는 설정 금지
- 예외: 매거진 서가의 세로 텍스트 (UI 요소이므로 별도)

### 5.4 한 줄 글자 수

- 본문(16px): 한 줄 35~45자 (한국어 기준, 720px 읽기 영역에서 자동 충족)
- 카드 제목: 2줄 이내로 truncate

## 6. 색상 팔레트

  --text-primary: #1A1A1A;      /* 제목, 본문 */
  --text-secondary: #6B7280;    /* gray-500 — 부제, 설명 */
  --text-tertiary: #9CA3AF;     /* gray-400 — 라벨, 메타 */

  --border: #F3F4F6;            /* gray-100 */
  --border-strong: #E5E7EB;     /* gray-200 — 구분선 */

  --accent: #4F46E5;            /* 인디고 — AI Insight, 특수 기능에만 */

### 6.2 다크 모드

모든 컴포넌트는 CSS 변수 사용 필수. 하드코딩 금지.

## 7. 간격 리듬 (8px 그리드)

모든 간격은 8의 배수로만 사용한다.

- 8px — 아이콘과 텍스트 사이, 아주 작은 gap
- 16px — 카드 내부 패딩, 작은 요소 간격
- 24px — 카드 간 gap, 중간 간격
- 32px — 섹션 내부 요소 간격
- 48px — 소형 섹션 간 여백
- 64px — 중형 섹션 간 여백
- 96px — 대형 섹션 간 여백 (기본)
- 128px — 페이지 최상단/최하단 여백

clamp 사용 시에도 상한/하한 모두 8px 배수여야 한다.

## 8. 모서리 (Border Radius)

### 8.1 규칙

| 요소 | radius | 비고 |
|------|--------|------|
| **블로그 카드** | 12px | 32~50px 금지 |
| **버튼** | 8px | pill(999px)은 특수 버튼에만 |
| **이미지** | 8px | 카드 안 이미지 |
| **모달/패널** | 16px | 대형 UI 요소 |
| **뱃지/태그** | 4px | 작은 라벨 |
| **서가 아이템** | 0px | 책 느낌 유지 |

### 8.2 원칙

- **32px 이상의 radius는 사용하지 않는다**
- 40~50px radius는 SNS/앱 스타일 → 에디토리얼과 맞지 않음
- 예외: 프로필 이미지 (원형 50% 허용)

## 9. 호버 & 트랜지션

### 9.1 카드 호버

```css
/* ❌ 금지 — 과도함 */
transform: translateY(-16px);
box-shadow: 0 30px 60px rgba(0,0,0,0.15);

/* ✅ 허용 — 절제 */
.card:hover {
  opacity: 0.92;
  transition: opacity 0.3s ease;
}
/* 또는 */
.card:hover .card-image {
  transform: scale(1.02);
  transition: transform 0.5s ease;
}
```

### 9.2 트랜지션 기본값

```css
transition: all 0.3s ease;
```

- cubic-bezier(0.16, 1, 0.3, 1)는 **서가 전용**으로만 사용
- 일반 UI 요소는 ease 또는 ease-out

### 9.3 애니메이션 원칙

- 페이지 로드 시 fade-in: opacity 0→1, 0.5s — 허용
- 스크롤 트리거 slide-up: **사용하지 않는다** (과도한 움직임)
- stagger 딜레이: **사용하지 않는다** (카드가 순서대로 튀어오르는 것은 게임 느낌)

## 10. 카드 디자인 규칙

### 10.1 블로그 카드 구조

```
┌─────────────────┐
│                  │
│   이미지 (3:4)   │
│                  │
│                  │
├─────────────────┤
│ CATEGORY         │  ← 11px, gray-400, uppercase
│ 제목 두 줄 이내   │  ← 18px, bold, black
│ (2줄 초과 시 ...)  │
└─────────────────┘
```

### 10.2 카드에 넣지 않는 것

- ❌ 날짜 (카드 레벨에서 불필요 — 상세 페이지에서 확인)
- ❌ 저자 (카드 레벨에서 불필요)
- ❌ 발췌문/요약 (제목으로 충분)
- ❌ 뱃지 (Featured는 그리드 배치로 구분, 별도 마크 불필요)
- ❌ 그라디언트 오버레이 (이미지 위에 텍스트 올리지 않음)

### 10.3 이미지 위 텍스트 금지 원칙

- 블로그 카드에서 이미지 위에 텍스트를 올리지 않는다
- 텍스트는 반드시 이미지 아래 별도 영역에 배치
- 예외: 히어로 캐러셀 (몰입형 섹션)

## 11. 매거진 서가 (시그니처 UI)

### 11.1 디자인 목표

> **"누군가 우리의 서가를 레퍼런스로 삼을 수 있을 정도"**

서가는 MHJ의 유일무이한 시그니처 UI다. 단순히 "잘 만든 슬라이더"가 아니라,
**서재에서 책등만 보이다가 한 권을 꺼내 펼치는 물리적 경험**을 재현해야 한다.

### 11.2 핵심 메타포: 세네카의 서재

- 접힌 상태(기본): **책등(spine)만 보인다**
  - 좁은 너비 (80~100px)
  - 세로 텍스트: 연도, 월, 제목
  - 어두운 톤, 텍스처 느낌
  - 서로 밀착되어 "서가에 꽂혀있는" 느낌

- 펼친 상태(호버/클릭): **책이 서가에서 살짝 빠져나와 표지가 보인다**
  - 부드럽게 확장 (320~480px)
  - 커버 이미지가 등장
  - 제목, 에디터, "Open Edition" CTA
  - 나머지 책들은 좁아지지만 사라지지 않는다

### 11.3 서가 트랜지션

```css
transition: width 0.8s cubic-bezier(0.16, 1, 0.3, 1);
```

- 이 이징은 서가 **전용** — 다른 곳에서 사용 금지
- 펼침 시 0.15s 딜레이로 커버 콘텐츠 fade-in (책이 열리는 타이밍)

| 일반 규칙 | 서가 예외 | 이유 |

## 15. 퍼블릭 vs 어드민

### 15.1 퍼블릭 수정 시 주의

- 어드민 작업 중 퍼블릭 페이지를 **절대 건드리지 않는다**
- 퍼블릭 레이아웃 변경은 **별도 세션에서 전용으로** 진행
- 변경 전 반드시 npm run build 성공 확인

### 15.2 폰트 크기 불가침 원칙

- **실제 폰트 크기를 줄이지 않는다**
- 미리보기/오버뷰용은 CSS transform: scale() 사용

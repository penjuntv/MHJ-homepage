## A. 전수하는 "스킬 구조" 패턴 (humanizer 계열에서 검증됨)

가사 에이전트가 헤매는 근본 원인은 대개 *지식 부족이 아니라 구조 부족*이다.
아래 5블록 골격을 스킬로 강제하면 출력 품질이 크게 안정된다.

1. **절대 규칙(Absolute rules)** — 타협 불가 제약. 매 출력에서 검사.
2. **제거할 흔적 블록리스트(AI tells)** — 추상적 "자연스럽게"가 아니라 *구체적 금지어/패턴 목록*.
3. **보이스 보존(Preserve the voice)** — 일반 다듬기가 놓치는 절반.
   클리셰만 빼면 다 똑같이 밋밋해진다 → 화자 고유 감정/질감을 의도적으로 지킨다.
4. **advisory 출력** — 자동 덮어쓰기 금지. before/after + 이유 표로 제시, 채택은 사람.
5. **Gotchas log** — 실패할 때마다 한 줄씩 누적해 스킬을 자가 진화시킨다.

> 핵심 교훈: "깨끗하지만 보이스 없는 가사"는 "티 나는 AI 슬롭"만큼 실패다.
> 모든 행을 같은 길이로 만들고 화자 감정을 중화시키면 과편집이다.

## B. 한국어 가사 도메인 지식 (블록 ②③에 채워 넣을 재료)

### B-1. 율격 / 음절수 — singability (가장 중요)
- 멜로디는 행당 음절수가 사실상 고정. 가사가 안 불러지는 1순위 원인 = 음절수 불일치.
- **멜로디/탑라인 있으면**: 각 행 목표 음절수를 받아 ±1 안에서 맞춘다.
- **가사 먼저면**: 후렴(hook) 음절수를 anchor로 고정하고 절을 거기에 맞춘다.
- 행 길이는 *일부러* 변주: 짧은 훅 행 + 긴 서술 행 대비가 리듬을 만든다.
  (전부 같은 길이 = 단조롭고 기계적)

### B-2. 운(韻) — 한국어식으로
- 영어식 강세운이 아니라 **모음운 + 받침운**으로 잡는다.
- 각운: 행 끝 모음/받침을 느슨하게 맞춤 (-아/-야, -어/-여, -ㅁ받침끼리 등).
- 완전운보다 **유사운**이 자연스럽다. 무리한 완전운은 오히려 번역투를 부른다.
- 후렴 끝 음절은 길게 끌리는 모음(아/오/이)이 발음·가창에 유리.

### B-3. 한국어 AI 가사 흔적 — 블록리스트 (실제 금지)
- **추상명사 슬롭**: 영원·운명·진심·우주·기적·완성·끝없이·존재·의미
- **번역투 구문**: "너의 ~가 나를 ~하게 해", 과도한 주어 명시("나는 너를…"),
  영어 어순 직역, 관계절을 명사 앞에 길게 붙이기
- **클리셰 비유**: 별처럼 빛나는 / 운명처럼 / 시간이 멈춘 듯 / 너는 나의 전부
- **감정 직설 과잉**: "너무 사랑해", "행복해" 등 *말하기*. 노래는 *보여주기*(장면).
- **조사 과잉**: 노래에선 조사 생략이 자연스럽다 ("내 맘을"→"내 맘", "너의"→"네").

### B-4. 번역투 진단 체크
- 주어가 매 행 등장하면 의심 (한국어는 주어 생략이 기본).
- "~것" 명사화 남발("사랑하는 것", "함께라는 것").
- 형용사+명사 직역("아름다운 순간들" 같은 무색 표현).
- 영어 원곡 번안이면, 직역 대신 **상황 재현(adaptation)** 으로 다시 쓴다.

### B-5. Hook / 후렴 설계
- 제목 구절을 후렴에 **반복 앵커**로 박는다 (청자가 따라 부를 지점).
- 후렴은 절보다 음절·모음이 단순·개방적이어야 입에 붙는다.
- 1절/2절은 후렴으로 수렴하는 빌드업. 절끼리는 구조(음절/운) 평행 유지.

### B-6. 화자 / 말투 일관성
- 시점(1인칭/2인칭/관찰자)·말투(반말/존댓말) 곡 전체 일관.
- 감정 아크: 1절 상황 → 2절 전개/반전 → 후렴 정서 고조. 행마다 흔들리지 않게.

### B-7. 발음 흐름 (가창성/낭송성)
- 자음 군집·받침 충돌 회피 (부르기 어려운 행 제거).
- 강세가 오는 음에 의미 있는 음절(주로 명사·동사 어간)이 오게.
- 소리 내어 읽어보는 셀프체크를 출력 전 1회 강제.

## C. 적용 워크플로우 (그쪽 에이전트가 쓰는 순서)
1. 입력 확인: 멜로디/탑라인 유무, 장르, 화자, 제목 구절(hook).
2. 후렴 음절수 anchor 결정 → 절을 거기 맞춤 (B-1).
3. 초안 생성 → B-3 블록리스트로 1차 세척.
4. B-2 운 / B-7 가창성 점검 (소리 내어 읽기).
5. B-5 후렴 앵커 / B-6 화자 일관성 확인.
6. before/after 표 + 이유로 제시 (advisory). 실패 사례는 Gotchas log에 적립.

## D. 경계 (하지 말 것)
- 화자 감정을 중화시켜 "무난한" 가사로 만들지 말 것 (보이스 소실 = 실패).
- 무리한 완전운·기교 운으로 의미를 비틀지 말 것.
- 사실/고유명사가 들어가면 그건 별도 fact 검증 영역(이 문서 범위 밖).
- 이 문서는 *재료*다. 최종 SKILL.md 형식·트리거·도구 권한은 받는 프로젝트가 정한다.
